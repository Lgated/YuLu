# 多租户智能客服中台 - 产品与技术架构设计

## 📋 一、现状分析

### 1.1 当前问题

**现状**：所有功能（AI对话、工单、通知）混在一个界面，没有区分用户角色和使用场景。

**问题**：
- ❌ 租户的**客户**（C端用户）不应该看到工单管理、数据看板等后台功能
- ❌ 租户的**管理员/客服**（B端用户）需要更专业的后台管理界面
- ❌ 用户体验混乱，功能边界不清晰
- ❌ 权限控制不够细化，安全性不足

### 1.2 目标用户群体

| 用户类型      | 角色    | 使用场景      | 核心需求                  |
| --------- | ----- | --------- | --------------------- |
| **C端用户**  | 租户的客户 | 咨询问题、寻求帮助 | AI对话、转人工、FAQ、满意度评价    |
| **B端管理员** | ADMIN | 管理租户、配置系统 | 数据看板、用户管理、系统配置、知识库管理  |
| **B端客服**  | AGENT | 处理工单、人工服务 | 工单系统、人工工作台、会话管理、知识库查询 |

#### 3.1.2 路由设计

```typescript
// src/routes/index.tsx
import { Routes, Route, Navigate } from 'react-router-dom';
import { getToken, getUserRole } from '../utils/storage';

// C端路由
const CustomerRoutes = () => (
  <Routes>
    <Route path="/customer/chat" element={<CustomerChatPage />} />
    <Route path="/customer/faq" element={<CustomerFAQPage />} />
    <Route path="/customer/history" element={<CustomerHistoryPage />} />
    <Route path="/customer/*" element={<Navigate to="/customer/chat" />} />
  </Routes>
);

// B端路由
const AdminRoutes = () => (
  <Routes>
    <Route path="/admin/dashboard" element={<AdminDashboardPage />} />
    <Route path="/admin/tickets" element={<AdminTicketListPage />} />
    <Route path="/admin/workspace" element={<AdminWorkspacePage />} />
    <Route path="/admin/sessions" element={<AdminSessionsPage />} />
    <Route path="/admin/*" element={<Navigate to="/admin/dashboard" />} />
  </Routes>
);

// 根路由 - 根据角色跳转
export default function AppRoutes() {
  const role = getUserRole();
  
  if (!getToken()) {
    return <Navigate to="/login" />;
  }
  
  // 根据角色跳转到对应门户
  if (role === 'USER') {
    return <CustomerRoutes />;
  } else if (role === 'ADMIN' || role === 'AGENT') {
    return <AdminRoutes />;
  }
  
  return <Navigate to="/login" />;
}
```
