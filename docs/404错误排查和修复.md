# 404错误排查和修复指南

## 🔍 问题分析

从错误信息看：
- **错误**：`POST http://localhost:5173/api/admin/auth/login 404 (Not Found)`
- **原因**：请求发送到了前端开发服务器（5173端口），而不是后端服务器（8080端口）

## ✅ 已完成的修复

### 1. 更新JWT过滤器排除URL配置

**文件**：`src/main/java/com/ityfz/yulu/common/config/WebMvcConfig.java`

**修改前**：
```java
registration.addInitParameter("excludedUrls",
    "/api/auth/login," +
    "/api/auth/registerTenant," +
    "/actuator/health," +
    "/error");
```

**修改后**：
```java
registration.addInitParameter("excludedUrls",
    "/api/admin/auth/login," +
    "/api/admin/auth/registerTenant," +
    "/api/customer/auth/login," +
    "/api/customer/auth/register," +
    "/actuator/health," +
    "/error");
```

## 🔧 可能的原因和解决方案

### 原因1：Vite代理配置未生效

**检查**：`frontend/vite.config.ts` 中的代理配置

```typescript
export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
});
```

**解决方案**：
1. **重启前端开发服务器**（重要！）
   ```bash
   # 停止当前服务器（Ctrl+C）
   # 重新启动
   cd frontend
   npm run dev
   ```

2. **检查后端服务是否运行**
   ```bash
   # Windows PowerShell
   netstat -ano | findstr :8080
   # 应该看到 LISTENING 状态
   ```

### 原因2：跨域配置问题

**检查**：`src/main/java/com/ityfz/yulu/common/config/WebMvcConfig.java`

跨域配置已正确：
```java
@Override
public void addCorsMappings(CorsRegistry registry) {
    registry.addMapping("/**")
        .allowedOriginPatterns("*")
        .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
        .allowedHeaders("*")
        .allowCredentials(true)
        .maxAge(3600);
}
```

**注意**：404错误通常**不是跨域问题**。跨域问题会返回CORS错误，而不是404。

### 原因3：后端路由未正确注册

**检查**：确认后端Controller是否正确

**B端登录**：`src/main/java/com/ityfz/yulu/admin/controller/AdminAuthController.java`
```java

public class AdminAuthController {
    @PostMapping("/login")

        // ...
```

**C端登录**：`src/main/java/com/ityfz/yulu/customer/controller/CustomerAuthController.java`

## 🧪 测试步骤

### 步骤1：测试后端接口（直接访问）

```bash
# 使用curl测试（PowerShell）
curl.exe -X POST http://localhost:8080/api/admin/auth/login `
  -H "Content-Type: application/json" `
  -d '{\"tenantCode\":\"TENANT_001\",\"username\":\"admin\",\"password\":\"123456\"}'
```

**预期结果**：
- 如果返回JSON（即使是错误），说明路由存在
- 如果返回404，说明路由不存在或后端未启动

### 步骤2：检查前端代理

1. 打开浏览器开发者工具（F12）
2. 切换到"网络"（Network）标签
3. 尝试登录
4. 查看请求详情：
   - **请求URL**：应该是 `http://localhost:5173/api/admin/auth/login`
   - **实际代理**：应该转发到 `http://localhost:8080/api/admin/auth/login`
   - **状态码**：如果是404，检查代理配置

### 步骤3：检查后端日志

查看后端控制台输出，确认：
- 请求是否到达后端
- 是否有路由匹配错误
- 是否有异常抛出

## 📝 常见问题

### Q1: 为什么是404而不是CORS错误？

**A**: 404表示路由不存在，通常是因为：
- 代理配置未生效（需要重启前端服务器）
- 后端路由未正确注册
- 请求URL拼写错误

### Q2: 代理配置正确，但还是404？

**A**: 检查：
1. 后端服务是否真的在运行
2. 后端Controller是否被Spring扫描到（检查包路径）
3. 后端是否有其他过滤器/拦截器阻止了请求

### Q3: 如何确认代理是否生效？

**A**: 
1. 查看浏览器Network标签中的请求详情
2. 查看后端日志，确认请求是否到达
3. 使用curl直接测试后端接口
